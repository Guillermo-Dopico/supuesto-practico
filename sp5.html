QUINTO SUPUESTO PRÁCTICO.

Descarga el archivo .aia importarlo en tu AppInventor y completa los siguientes puntos. Al terminar vuelve a descárgalo y entrégalo en classroom.

    SCREEN 1 (1p)

  • Añade cuatro botones en una disposición parecida. Cuando se pulse el botón 3 justo después de haber pulsado el botón 1 abrimos la Screen_Juego. En cualquier otro caso no hacemos nada.

    • SCREEN JUEGO (6p)

  • Al cargar la pantalla el botón azul se mueve hacia la derecha desde un punto aleatorio de la parte superior. El botón rojo de forma análoga se desplaza hacia la izquierda desde un punto aleatorio desde la parte inferior.
  • Los botones rebotan al chocar los bordes de forma que el rojo se mueve de derecha a izquierda de forma constante. El botón azul se comporta de similar.
  • El label_boton_azul aparece el texto Botón azul aparece el texto Botón azul 0 punto Este texto se actualiza sumando 1 punto cada vez que se pulsa el botón azul. El label_boton_rojo funciona de forma análoga.
  • Si la puntuación de alguno de los dos botones llega a 15 entonces se abre la screen_final.
  • Tienes que guardar la punción de cada uno de los botones en una base de datos antes de cambiar de pantalla.

    • SCREEN FINAL (3p)

  • Al cargar la pantalla se recupera de la base de datos las puntuaciones de cada uno de los botones y se muestran el mensaje que corresponde el label adecuado.
